# Checklist Breve de Pruebas y Entrega

## 1) Checklist de pruebas mínimas
- [ ] El programa corre al menos una vez con entrada válida.
- [ ] Se probó al menos una entrada inválida.
- [ ] Se registró resultado esperado vs obtenido.
- [ ] Se documentaron errores y su manejo.

## 2) Checklist de documentación
- [ ] `docs/propuesta.md` completo.
- [ ] `docs/caso_de_uso.md` completo.
- [ ] `docs/estructura_repositorio.md` revisado.
- [ ] `docs/plan_de_pruebas.md` con al menos 3 casos.

## 3) Checklist de ejecución
- [ ] Existe `src/main.<ext>`.
- [ ] `scripts/run.sh` ejecuta o guía la ejecución.
- [ ] El proyecto no depende de servicios externos.

## 4) Checklist de entrega final
- [ ] Repositorio ordenado y sin archivos basura.
- [ ] Alcance pequeño y bien justificado.
- [ ] README actualizado.
- [ ] Entrega lista en GitHub Classroom.
