# Actividad: Diseño de Propuesta de Práctica Temática (Proyecto Pequeño)

## Descripción general
En esta actividad de GitHub Classroom vas a diseñar una **propuesta técnica pequeña** para resolver una necesidad real con enfoque en terminal y programación de sistemas. Primero debes **documentar y justificar** tu idea; después, de manera opcional, puedes implementar un prototipo mínimo.

## Objetivo de aprendizaje
Al finalizar, el estudiante será capaz de:
- Delimitar el alcance de una práctica pequeña y viable.
- Justificar la elección de lenguaje según el problema.
- Diseñar la estructura de un repositorio limpio y mantenible.
- Documentar caso de uso, pruebas y criterios de éxito antes de programar en grande.

## Lenguajes permitidos
- ARM64 Assembly
- C
- Python
- Bash

## Reglas para mantener el proyecto pequeño
- Máximo 1 problema principal por resolver.
- Máximo 1 archivo principal de ejecución (`src/main.<ext>`).
- Evita dependencias externas complejas.
- No usar frameworks, bases de datos, Docker, nube ni APIs externas.
- Si eliges ARM64 Assembly, limita el proyecto a operaciones muy básicas de consola y cálculo simple.

## Entregables esperados
1. Propuesta completa en `docs/propuesta.md`.
2. Caso de uso completo en `docs/caso_de_uso.md`.
3. Justificación de estructura en `docs/estructura_repositorio.md`.
4. Plan de pruebas en `docs/plan_de_pruebas.md` y `tests/test_plan.md`.
5. (Opcional) Prototipo mínimo funcional en `src/main.<ext>`.

## Instrucciones para el estudiante
1. Lee todo el repositorio antes de escribir código.
2. Completa primero los documentos de `docs/`.
3. Define un alcance mínimo realista (que puedas terminar en poco tiempo).
4. Si implementas prototipo, que sea corto, claro y ejecutable en terminal.
5. Registra pruebas manuales con evidencia simple (texto de salida y estado).

## Criterios generales de evaluación
- Claridad del problema y objetivo.
- Coherencia entre lenguaje elegido y solución propuesta.
- Delimitación correcta del alcance (sin sobreingeniería).
- Calidad de documentación técnica.
- Calidad mínima del prototipo (si se incluye).
- Organización del repositorio.

## Temas ejemplo (elige uno o inspira el tuyo)
- **Mini Toolkit en ARM64**
- **Asistente de Estudio en Terminal**
- **Reporteador de Información del Sistema**
- **Organizador de Archivos**
- **Juego de Aprendizaje en Línea de Comandos**

## Nota importante
En esta actividad, **primero se documenta y justifica la propuesta**. La implementación de código es **secundaria y opcional** como prototipo pequeño.
